## ⚠️ 残る問題（未解決）

### 5. EPIPE クラッシュ — Reed の並行フェッチで未解決
- **現象**: 3回目の並行フェッチで Node.js Pipe Transport が EPIPE (-32) でクラッシュ
- **発生箇所**: `scraper_reed.py` の `fetch_descriptions_concurrently()`（`scraper_helper.py`）
- **concurrency=2 でも解消せず**: 以前は4→2に減らしたが、3回の並行フェッチ目で再発
- **根本原因**: Playwright の Python ↔ Node.js Pipe Transport が、複数ページを同時に開閉する際の書き込み競合に弱い。Python側の `try/except page.close()` では捕捉できない（Node.js 側で例外発生）
- **本当の直し方**: 複数ページを同時に開くのではなく、**1つのページを再利用**して逐次 fetch する。ページを作り直さないのでパイプへの書き込み競合が原理的に起きない。`fetch_descriptions_concurrently` を `fetch_descriptions_sequential` に置き換えるか、`asyncio.Semaphore` を 1 に設定する

### 6. Guardian scraper — inline script 実行で `p` が None
- **現象**: `AttributeError: 'NoneType' object has no attribute 'chromium'`
- **発生条件**: `scraper_guardian.py` を直接 `python -c` や別スクリプトから import して async 実行した場合
- **原因**: `scraper_guardian.py` のモジュールレベルで `p = None` として宣言しておき、関数内で `async with async_playwright() as p:` で初期化するパターンになっているが、特定のコードパスで初期化前に `p.chromium` を参照している
- **影響**: `run.py` 経由では問題なく動作（run.py が playwright の初期化を正しく行う）。独立呼び出し時のみ発生

### 1. Ollama 404 エラー — analyzer が `gemma4:26b` を使えない
- **現象**: `http://localhost:11434/api/chat` が 404 Not Found
- **Ollama 自体は稼働中**: `curl http://localhost:11434/api/tags` で `gemma4:26b` がリストされる
- **影響**: `analyzer.py` が Ollama でジョブ詳細を解析できない → フォールバック推論（低品質）で動いている
- **原因未特定**: モデル名の解決問題か、`/api/chat` エンドポイントの仕様変更か
- **確認すべき**: `curl http://localhost:11434/api/chat -d '{"model":"gemma4:26b","messages":[{"role":"user","content":"hello"}]}'` を手動で実行して 404 が再現するか
